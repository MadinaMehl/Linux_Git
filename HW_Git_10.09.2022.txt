Укажите следующие команды:

1.  Для полной очистки индекса
    git reset

2. Для просмотра разницы между рабочим каталогом и последним коммитом
    git diff HEAD

3. Для создания новой ветки и одновременного переключения в на нее
    git checkout  -b  bugfix

4. Для сравнения веток между собой
    git diff  ..bugfix
    git diff master  bugfix

5. Для удаления локальной ветки
    git branch -D fixbug
6. Для слияния веток
    Необходимо находиться на основной ветки мастер и затем git merge  fixbag

7. Для перезаписи последнего коммита
    git commit  --amend  -n " ...."  при условии если коммит не был запушен.

Коротко опишите причину возникновения конфликтов при слиянии и порядок действий 
их решения.

Обычно конфликты возникают, когда два человека изменяют одни и те же строки в файле 
или один разработчик удаляет файл, который в это время изменяет другой разработчик.
Есть 2 сценария решения этой проблемы:
1. Прервать слияние веток.
    git merge  --abort
2. Вручную отредактировать файл с конфликтом.
